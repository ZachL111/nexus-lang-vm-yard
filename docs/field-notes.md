# Field Notes

`nexus-lang-vm-yard` is easiest to review by starting with the fixture, not the prose.

The domain cases cover `IR pressure`, `lowering drift`, `stack depth`, and `diagnostic reach`. They sit beside the smaller starter fixture so the project has both a compact scoring check and a domain-flavored review check.

The model rewards signal and confidence, then pulls back for drag. In this fixture, `stale` shows where that pullback matters most.

The extra check gives the repository a behavior path that can fail for a domain reason, not only a syntax reason.
